# /retrospective — Sprint Retrospective

## Trigger
Run at the end of each sprint.

## Flow
1. **Gather data**: Completed tasks, bugs found/fixed, velocity metrics
2. **What went well**: List wins and successes
3. **What didn't go well**: List problems and friction points
4. **Action items**: Specific, assignable improvements for next sprint
5. **Save**: Write to production/retro-sprint-{N}.md

## Output
- Retrospective document with metrics
- Prioritized action items with owners
- Updated velocity baseline
